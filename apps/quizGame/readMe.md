# Quiz App

A responsive mobile-first Quiz App built with HTML, CSS (BEM methodology), and vanilla 
JavaScript. Features include question browsing, bookmarking, answer toggling, 
user-generated content via a form, and basic profile management.

## Overview

The app consists of three main pages:

- **Home Page**  
  The home page displays all questions available on the platform.  
  Users can scroll through the list of quiz cards to explore the full question pool.

- **Bookmark Page**  
  The bookmark page shows only the questions that have been bookmarked.  
  It works like a favorites list, allowing quick access to saved questions.

- **Add Page**
  The add page features a form to create new questions.
  Users can input questions, answers, and tags, with live character counters and dynamic card creation.

- **Profile Page**  
  The profile page lets users view and manage their profile information and settings.
  A dark mode toggle is integrated (bonus feature)

## Project Goals

- **Core Goals**
  Create a responsive, mobile-first Quiz App with HTML, CSS (BEM methodology), and vanilla JavaScript. 
  The app includes four pages: Home (browse all questions), Bookmarks (marked questions), Add (add new 
  questions), and Profile (user data with Dark Mode).

- **Layout Focus** (Project 1)
  Build the basic structure with three/four pages, use Flexbox/Grid for cards, navigation, and header. 
  Implement consistent design with zoo-like, quiz-show style (blue backgrounds, banners with animals).

- **Interactivity** (Project 2+)
  Add bookmarking, answer toggle (show/hide), form validation, and Dark Mode (profile-only). Use event 
  delegation, DOM manipulation, and data attributes for dynamic content.

- **Technical Requirements**
  Semantic HTML with accessibility (ARIA), BEM classes, event handling, and responsive design. Later 
  extensions: Full Dark Mode, user-generated content.

## Technologies Used

- **HTML5** (semantic elements)
- **CSS3** (BEM, custom properties for dark mode, transitions)
- **Vanilla JavaScript** (DOM manipulation, event listeners, no frameworks)
- **Git/GitHub** for version control
