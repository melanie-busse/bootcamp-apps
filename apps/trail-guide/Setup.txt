4. Umgebungsvariablen (.env)
Erstelle eine Datei namens .env im Hauptverzeichnis. Hier kommen sensible Daten oder Pfade rein, die nicht direkt im Code stehen sollten:

Code-Snippet
PORT=3000
DB_PATH=./data/trail-guide.db
API_KEY=dein_geheimer_super_schluessel
Wichtig: Erstelle auch eine .env.example mit den gleichen Schlüsseln (aber ohne echte Werte) und füge .env zu deiner .gitignore hinzu, damit dein API-Key nicht auf GitHub landet.
